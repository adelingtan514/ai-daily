从 Gemini 2.5 到 Cursor 自研：2026 年，AI 产品公司的分水岭是「拥有模型」还是「调用模型」

事实
  • Gemini 2.5 Pro 在 LMArena 以 1443 分断层领先 GPT-5.4 和 Claude Opus 4.6（Google DeepMind，2026-03-26）；蚂蚁 F2LLM-v2 在 MTEB 拿下 11 项冠军，超越所有开源闭源对手——同一周内，两个不同维度的 AI 能力标杆被同时刷新，顶层模型格局正在加速分化。
  • Cursor 不再依赖 GPT/Claude API，转而发布自研 Composer 2：20 万 Token 上下文、SWE-bench 多语言指标全面提升、定价自主（百万 Token 输入 $0.50）——这是一个顶级 AI 产品公司用行动宣告「API 依赖是长期天花板」的里程碑（Cursor 官方，2026-03-19）。

观点

本周最值得关注的信号不是某个分数刷新，而是一个结构性转变：AI 行业正从「谁调用最强模型」演变为「谁拥有自己的模型」。Cursor 的自研 Composer 2 不只是一次技术升级，更是一次战略宣言——任何依赖第三方模型 API 的产品公司，最终都将面临能力上限锁死和定价权丧失的双重困境。而 Gemini 2.5 Pro 的断层优势和蚂蚁 F2LLM-v2 的 MTEB 全面突破，印证了另一趋势：AI 能力正在按维度分化，推理、编程、检索将由不同专家模型主导，「最强单模型」的叙事正被「能力矩阵」取代。对普通人来说，这意味着工具选择越来越重要——用对工具的人正在以指数级速度拉开与用错工具的人的差距。而 GitHub Copilot 的数据争议则提醒我们：工具越好用，越要审视它在用什么换取这份便利。

自研模型是 2026 年 AI 产品的分水岭：API 调用方终将面临能力天花板和定价锁死——Cursor 的选择已经给出了最清晰的答案。
